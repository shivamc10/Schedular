    Security and compliance - 58 points

    • Finish Any CCPA Related Work for 7/1/20 Deadline (Downstream Opt-Outs, Fulfillment, etc.)

      • We are done with Centralized Opt-Out implementation for Match Extract Partners (including xref-producer) - Completed

    • Encrypting data in transit - 3 points*
      • We worked on it for PII(first_name, last_name, address1 etc..) during FY20.Q2, - 3 points*
      • Question - do we require to work on this for PI(hhid, ind_id etc..) during FY20.Q3 ? - 
        • Currently we are only focused PII (First Name, Last Name, Address, etc.). ID's are not of concern at this point in time. It is not a bad thing for us to do at some point in time but we likely have more pressing work curently. 
    • Qualys and CVE Remediation before 2/29/20 - 5 points
      • We require to work on the remaining git projects for Match Extract and Hyperloop - 5 points
    • Storing the credentials in Vault for Hyperloop and Match Extracts - 50 points
      • Hyperloop has usernames and passwords
      • Match Extracts has keys for encryption
      • Question - Is this part of Security and compliance?
        • TS- Yes, I think it is good to add it here. When we talk about creating tooling around making the creation and storage of credentials easier/more secure we may want        to relate that to "First Class Services". But the act of moving the credentials into Vault is more focused on immediate improvement of Security and Compliance.
      • Stories:
        • Spike on Vault usage: - 5 points
          • To find a way of storing shared credentials at one place and find alternative of twinfin - 3 points
          • To find out if two projects can be merged to one and implement vault - 2 points
          •  13-Gradle, 20-Makefile projects and 1 DAC => 13*1 + 17*1.5 = ~45 points (Committing for 22 out of 30 projects)

    Reducing Risk of Data Misuse

    • Shared Core Services has record of All Files Delivered to ODC

      • Question - 
        • do we consider sending data to external partners?
          • TS - Right now we are more focused on what is coming into our company. This is really a goal for the DÜM team to begin to capture an inventory of all datasets. So when we begin to create more Data Lineage we are able to have the "First Scene Instance" of all data being sent to us.
        • Should we consider data copying in Hyperloop as file delivered?  
          • TS - We should have a log of all copies Hyperloop is performing. This does not need to be integrated with other services necessarily at this point in time, but we should know : What Jobs are Setup? Who owns them? How often should they run? When was the last time they ran? - Jobs catalog has the same info

    • Data Catalog Integrates with Data Lineage Service

      • Question - 
        • Are we going to get more requests on integration of any other data flows through Hyperloop?
          • TS - We are currently pausing on more Data Lineage work right now in order to focus on capturing Data Inventory (All Inbound Files). So we should not expect any new integration adjustments/asks this quarter. However, if we are setting up new Hyperloop syncs, we likely want to include the current Data Lineage Integration code Robert had requested to be included.
